What is BDD & Cucumber?
BDD — Behavior Driven Development
Tests written in plain English that everyone can understand — developers, testers, and business people.

Tests describe the behavior of the app, not the code. Written before coding starts to bridge the gap between technical and non-technical teams.
Cucumber JS
A BDD framework that reads plain English test scenarios and executes them as automated tests.

Your existing Playwright framework can be converted to BDD by adding the @cucumber/cucumber package and writing .feature files.
💬 In short: BDD is the approach — Cucumber is the tool that implements it.

Step Definitions
How to get step stubs
Run npx cucumber-js and Cucumber will output the missing step definitions. Copy them into a file under features/support/ (e.g. login.js).
1. Add this import at the very top of your file:
import { Given, When, Then } from "@cucumber/cucumber";
2. Paste the generated step stubs and fill in the automation code:
Given('User is in login page of Application', function () { // Write code that navigates to the login page return 'pending'; }); When('User Enter valid User id and valid password', function () { // Write code to fill in credentials return 'pending'; }); When('Click on login button', function () { // Write code to click the login button return 'pending'; });

CLI Commands
Run all tests
$ npx cucumber-js
Run a specific feature file
$ npx cucumber-js features/login.feature
Load step definitions from a path
$ npx cucumber-js --require features/step_definitions/**/*.js
Run by tag
$ npx cucumber-js --tags "@login"
Tag — AND condition
$ npx cucumber-js --tags "@smoke and @login"
Tag — OR condition
$ npx cucumber-js --tags "@smoke or @regression"
Tag — NOT condition
$ npx cucumber-js --tags "not @wip"
Dry run — check for missing step definitions
$ npx cucumber-js --dry-run
Run a specific feature folder
$ npx cucumber-js features1/

Cucumber Flags Reference
Flag Purpose
features Path to the feature folder containing .feature files
--require Path of step definitions (glue code) — tells Cucumber where to find them
--dry-run Checks whether all steps have matching step definitions — lists missing ones without actually running tests
--tags Run selective tests by tag. Supports and, or, and not logic
--format Generate reports in different formats: json, xml, html

How Cucumber Executes Tests
1
Scans the features/ folder and picks up all .feature files
2
Reads each scenario one by one — each scenario is an independent test case
3
For each step (Given / When / Then), it looks up the matching step definition in the support folder
4
Executes the step definitions sequentially, reporting pass, fail, or pending for each step
